# Commerce AdPilot MVP UI Wireframe

이 문서는 PRD와 사용자 시나리오(마케팅 팀장 중심 운영 흐름)를 기준으로 작성한 저충실도(low-fidelity) 와이어프레임입니다.
목표는 로그인 없는 즉시 사용, 대량 CSV 업로드, 멀티 채널 동시 생성, 빠른 편집/복사/내보내기 흐름을 화면 구조로 구체화하는 것입니다.

## 1) MVP 설계 원칙

- 로그인 없이 즉시 진입하고 바로 생성 작업 시작
- 상품 1건 수기 입력과 다건 CSV 업로드를 같은 화면에서 지원
- 블로그/인스타그램/페이스북 카피를 한 번에 동시 생성
- 생성 결과는 채널별 미리보기와 인라인 편집 중심으로 검수
- 복사/텍스트 내보내기까지 단일 흐름으로 완료
- 목표 성능: 초기 결과 10초 이내, 상품군 단위 작업 10분 내

---

## 2) IA (Information Architecture)

- 랜딩/진입: 제품 소개, 즉시 시작 CTA, 언어 선택(KR/EN)
- 생성 워크스페이스(핵심 단일 페이지):
	- 입력 영역: 수기 입력 + CSV 업로드 + 팀 가이드(톤/핵심 메시지/금지어)
	- 생성 옵션 영역: 채널 선택, 톤앤매너, 길이/목적, 재생성 제어
	- 결과 영역: 상품별/채널별 결과, 인라인 편집, 복사, 파일 내보내기
- 상태/피드백: 로딩, 부분 실패, 재시도, 완료 요약

---

## 3) 주요 사용자 플로우

1. 대표 페이지 접속 -> 로그인 없이 시작 -> 생성 워크스페이스 진입
2. 팀 가이드 입력(기본 톤, 핵심 메시지, 금지/필수 표현)
3. 상품 데이터 입력(수기 또는 CSV 업로드)
4. 채널/톤 옵션 선택 후 동시 생성
5. 채널별 결과 검토 및 인라인 편집, 필요한 항목만 재생성
6. 확정본을 클립보드 복사 또는 TXT/CSV로 내보내기

시나리오별 공통 변형
- 긴급 캠페인: 기존 CSV 재업로드 -> 행동 촉구 톤으로 일괄 생성 -> 기간/할인 문구 빠른 보정
- 신규 카테고리: 전문적/감성적 톤 비교 생성 -> 최적 문체 확정
- 저성과 리프레시: 기존 톤 변경 재생성 -> CTA 중심 수정 후 즉시 교체 배포

## 4) 데스크톱 와이어프레임

### 4.1 즉시 진입 화면 (로그인 없음)

```
+--------------------------------------------------------------------------------+
| Commerce AdPilot                                             [KR|EN] [바로 시작]|
|--------------------------------------------------------------------------------|
| "상품 정보 한 번 입력으로 채널 카피를 동시에 생성"                                |
|                                                                                |
| [핵심 가치]                                                                     |
| - 평균 작업시간 1시간 -> 10분 수준 단축                                         |
| - 블로그/인스타그램/페이스북 동시 생성                                          |
| - 편집/복사/내보내기 원스톱                                                     |
|                                                                                |
|                            [지금 생성 시작하기]                                 |
+--------------------------------------------------------------------------------+
```

요소 메모
- 인증 관련 UI 제거
- 첫 진입에서 기능 설명보다 즉시 작업 시작 CTA를 우선 배치

### 4.2 생성 워크스페이스 (핵심)

```
+------------------------------------------------------------------------------------------------+
| Logo | 콘텐츠 생성 워크스페이스                                      상태: 대기                  |
|------------------------------------------------------------------------------------------------|
| [좌측: 입력]                                      | [우측: 생성 옵션]                          |
|------------------------------------------------------------------------------------------------|
| 입력 방식: (o) 수기 입력  ( ) CSV 업로드          | 생성 채널*                                  |
|------------------------------------------------------------------------------------------------|
| 상품명*          [____________________________]    | [x] 블로그  [x] 인스타그램  [x] 페이스북   |
| 카테고리*        [____________________________]    |                                              |
| 주요 특징*       [+ 특징 추가]                    | 톤앤매너* [전문적 v]                        |
| 타겟 고객*       [____________________________]    | 길이      [표준 v]                           |
| (선택) 가격대    [____________________________]    | 목적      [전환 v]                           |
|------------------------------------------------------------------------------------------------|
| 팀 가이드(공통 프롬프트)                           | [생성하기] [선택 항목 재생성] [전체 초기화] |
| 기본 톤         [친근하면서도 프로페셔널________]   |                                              |
| 핵심 메시지      [고품질/합리적 가격/빠른 배송___]   | 예상 생성 시간: 약 5~10초                    |
| 금지 표현        [____________________________]    | 실패 항목: 0건                               |
| 필수 표현        [____________________________]    |                                              |
|------------------------------------------------------------------------------------------------|
| CSV 업로드 패널(토글)                                                                      |
| [파일 선택] sample_products.csv   [업로드] [템플릿 다운로드]                               |
| 매핑: 상품명/카테고리/주요특징/타겟고객                                                   |
| 미리보기: 총 200건 중 유효 196건, 오류 4건 [오류 행 보기]                                  |
+------------------------------------------------------------------------------------------------+
```

요소 메모
- PRD 필수 항목(상품명/카테고리/주요 특징/타겟 고객)을 기본 검증 필드로 고정
- CSV는 업로드 즉시 유효성 체크와 행 단위 오류 피드백 제공
- 팀 가이드를 공통 프롬프트로 분리해 브랜드 일관성 확보

### 4.4 결과 검토/편집 화면

```
+------------------------------------------------------------------------------------------------+
| 생성 결과 (총 200건)                                                [전체 내보내기] [전체 복사] |
| 필터: [전체 v] [채널 v] [완료/실패 v] [검색: 상품명______________]                      |
|------------------------------------------------------------------------------------------------|
| 상품: 프리미엄 무선청소기 A100  | 상태: 완료 | [해당 상품 재생성]                              |
|------------------------------------------------------------------------------------------------|
| 블로그                                                                         [복사] [다운로드] |
| 원본: 3~5문장 결과 텍스트...                                                                  |
| 수정: [인라인 편집 텍스트영역.........................................................]       |
|------------------------------------------------------------------------------------------------|
| 인스타그램                                                                     [복사] [다운로드] |
| 원본: 짧은 후킹 + 해시태그...                                                                 |
| 수정: [인라인 편집 텍스트영역.........................................................]       |
|------------------------------------------------------------------------------------------------|
| 페이스북                                                                       [복사] [다운로드] |
| 원본: 정보형 카피 + CTA...                                                                     |
| 수정: [인라인 편집 텍스트영역.........................................................]       |
|------------------------------------------------------------------------------------------------|
| [변경 저장] [선택 채널만 재생성] [다음 상품 >]                                                   |
+------------------------------------------------------------------------------------------------+
```

요소 메모
- "원본 vs 수정본" 동시 노출로 검수 효율 향상
- 채널별 독립 재생성 버튼 제공(전체 재생성 최소화)
- 복사와 다운로드를 카드 단위 + 전체 단위 모두 지원

## 6) 컴포넌트 체크리스트 (MVP)

- 즉시 진입 Hero + CTA 컴포넌트
- 수기 입력 폼 + 실시간 유효성 검사
- CSV 업로드/파싱/행 오류 테이블 컴포넌트
- 채널 멀티셀렉트 + 톤앤매너 옵션 컴포넌트
- 생성 상태 컴포넌트(진행률/부분 실패/재시도)
- 결과 카드 컴포넌트(채널별 원본/수정본/복사/재생성)
- 내보내기 모달(TXT/CSV, 범위 선택)
- 공통 알림 컴포넌트(성공/실패/제한 안내)

---

## 7) 접근성/국제화/운영 메모

- 모든 입력 필드 label 연결 및 오류 메시지 ARIA 속성 제공
- 키보드만으로 생성/편집/복사/내보내기 가능해야 함
- 상태 정보는 색상 + 텍스트 + 아이콘 동시 제공
- KR/EN 문자열 리소스 분리, 채널명/톤 옵션도 로케일 대응
- 성능 지표 이벤트 수집 권장:
	- 생성 시작/완료 시간
	- 상품군 단위 완료 시간
	- 실패율 및 재시도 성공률

---

## 8) 범위 정렬 체크 (PRD/시나리오 대비)

- 포함: 로그인 없는 진입, CSV 대량 업로드, 멀티 채널 동시 생성, 편집/복사/내보내기
- 제외: 회원가입/로그인, 프로젝트 대시보드, 계정 설정, 외부 플랫폼 직접 연동
