# Commerce AdPilot MVP UI Wireframe

이 문서는 PRD 기반의 저충실도(low-fidelity) 와이어프레임입니다.
목표는 화면 구조, 정보 우선순위, 핵심 사용자 흐름을 빠르게 합의하는 것입니다.

## 1) IA (Information Architecture)

- 인증: 회원가입 / 로그인 / 로그아웃
- 워크스페이스: 프로젝트(캠페인) 목록, 최근 작업
- 콘텐츠 생성: 제품 정보 입력 + 생성 옵션 + 결과 섹션
- 콘텐츠 편집/저장: 원본/수정본 비교, 저장, 복사, 다운로드
- 이력: 시간순 생성 이력, 재열기, 재사용
- 설정: 기본 언어, 기본 톤, 기본 출력 형식

---

## 2) 주요 사용자 플로우

1. 로그인 -> 프로젝트 선택/생성 -> 제품 정보 입력 -> AI 생성 -> 편집 -> 저장/내보내기
2. 이력 페이지 -> 기존 생성 건 열기 -> 재편집 -> 재저장
3. 생성 실패 -> 원인 확인(입력/서버/요청 제한) -> 재시도

## 3) 데스크톱 와이어프레임

```
+--------------------------------------------------------------------------------+
| Commerce AdPilot                                                       [KR|EN] |
|--------------------------------------------------------------------------------|
|                                                                                |
|                         +------------------------------+                       |
|                         |          로그인              |                       |
|                         |------------------------------|                       |
|                         | 이메일      [______________] |                       |
|                         | 비밀번호    [______________] |                       |
|                         | [ ] 로그인 상태 유지         |                       |
|                         |                              |                       |
|                         | [ 로그인 ]                  |                       |
|                         |                              |                       |
|                         | 회원가입하기  비밀번호 찾기  |                       |
|                         +------------------------------+                       |
|                                                                                |
+--------------------------------------------------------------------------------+
```

요소 메모

- 상단 우측 언어 토글(KR/EN)
- 오류 메시지는 필드 하단 인라인 표기

```
|--------------------------------------------------------------------------------|
```

```
|--------------------------------------------------------------------------------|
| 제품 이름*        [__________________________]  | 언어      [한국어 v]           |
| 카테고리*         [__________________________]  | 톤앤매너  [친근한 v]           |
| 주요 특징*        [+ 특징 추가]                | 길이      [표준 v]             |
|  - [______________________________]            | 목적      [전환 v]             |

| 가격대(선택)      [__________________________]  | [생성하기] [전체 초기화]       |
| 브랜드 톤(선택)   [__________________________]  |                                  |
| 금지어(선택)      [__________________________]  | 상태: 대기 / 생성중 / 실패      |
```

## 5) 컴포넌트 체크리스트 (MVP)

- 필수 입력 검증 컴포넌트 (실시간)
- 생성 옵션 선택 컴포넌트 (언어/톤/길이/목적)
- 결과 섹션 카드 (원본/수정본/재생성)
- 공통 액션 버튼 (복사/다운로드/저장)
- 상태 표시 컴포넌트 (로딩/실패/재시도)
- 이력 테이블 + 상세 열기
- 설정 폼 + 저장 피드백

---

## 6) 접근성/국제화 메모

- 모든 입력 필드에 label 연결 및 오류 텍스트 ARIA 연계
- 키보드만으로 생성/편집/저장 가능해야 함
- 색상 외에 아이콘/텍스트로 상태 구분
- 한국어/영어 문자열 리소스 분리(i18n 키 구조 권장)
