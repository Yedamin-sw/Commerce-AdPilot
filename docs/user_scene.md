# 사용자 시나리오 (User Scene)

## 1. 판매자 시나리오
- 제품명, 카테고리, 주요 특징, 타겟 고객 입력
- 광고 문구, 제품 설명, SNS 포스트, 해시태그 생성 요청
- 생성 결과 미리보기, 편집, 저장, 다운로드

## 2. 마케터 시나리오
- 스타일(톤, 길이)을 선택한 콘텐츠 생성
- 짧은 버전/긴 버전 비교
- 여러 캠페인 콘텐츠 생성 후 클립보드 복사, 저장

## 3. 초보 판매자 시나리오
- 최소 정보 입력으로 빠른 콘텐츠 생성
- 한글/영어 다국어 옵션 사용
- 생성 이력에서 이전 콘텐츠 재사용

## 4. 계정 기반 시나리오
- 회원가입/로그인(Supabase Auth)
- 생성 이력 조회, 기본 설정 관리
- 생성 횟수 및 최신 생성 내역 확인

## 5. 상세 시나리오 케이스
### Case A: 신제품 런칭 광고 문구 생성
1. 사용자: 판매자 A
2. 입력: 제품명(무선 이어폰), 카테고리(전자), 주요 특징(노이즈 캔슬링, 30시간 배터리), 타겟(젊은 직장인)
3. 요청: 짧은 광고 문구 3개, 긴 광고 문구 2개, 해시태그 제안
4. 검증: 생성된 문구가 특징/타겟 반영, 1분 이내 응답
5. 액션: 마음에 드는 문구 선택 후 복사/다운로드

### Case B: 캠페인 콘텐츠 스타일 변환
1. 사용자: 마케터 B
2. 입력: 기존 설명 텍스트 (제품 기본 정보)
3. 옵션: 톤(친근), 길이(짧음), 언어(영어)
4. 요청: SNS 포스트 5개, 해시태그 10개
5. 검증: 스타일(친근), 짧은 형태, 다국어 여부 확인

### Case C: 이력 재사용 & 편집
1. 사용자: 초보 판매자 C
2. 과거 생성 이력에서 선택(6일 전 생성한 제품 설명)
3. 요청: 표현 일부 수정 + 업데이트 저장
4. 검증: 이력 수정 사항 DB 반영, 버전 기록 유지

### Case D: 잘못된 입력 처리
1. 사용자: 누구나
2. 입력: 제품명 누락, 또는 타겟 고객 빈 값
3. 요청: 콘텐츠 생성 시도
4. 검증: `400 Bad Request` + 필수 항목 안내 메시지 표시
5. 액션: 항목 보완 후 재시도
